HCA User Interface
The HCA UI is a work in progress, not yet ready for practical use.
HCA stands for Heterogeneous Computing Architecture. HCA_UI is a user interface to writing software for Dexter's FPGA, using a graphical editor. The tool creates a JSON representation of the layout of the objects in the editor. That JSON representation is sent to a compiler that makes a compiled gateware file which is stored on Dexter and controls the computation of the FPGA. That gateware can move Dexter's joints, captures joint positions from the optical sensor on each joint and perform additional very fast. parallel computations.

DDE User Interface for HCA The HCA UI reuses components of the DDE user interface for those opperations that are in common with normal DDE editing.
  • Choosing FPGA Programming from the list of tutorials shown when clicking the button displays this documentation in the Doc pane.
  • Choosing from the Editor pane's view menu, shows the HCA blocks-and-wires graph editor.
  • Choosing from the Editor pane's view menu, while viewing HCA, writes out HCA JSON to the editor pane, with the help of a dialog box to give you options on what Object Definitions to put into the text editor pane.
  • Clicking with JSON in the editor pane will inspect the JSON, particularly useful for the large amount of JSON generated by HCA, to make it easier to understand the hierarchy.
  • File menu/New Clears the editor of all blocks to start a new object definition.
  • File menu/Open ... Allows the user to select a file from the computer's file system to edit.
  • Edit menu/pretty print arranges the blocks to be well spaced.
  • Find When the HCA view is shown in the Editor pane, you can use the type-in and button at the top of the Doc pane to search for HCA definitions. The results are shown in the inspector for direct matches, search term included in name, and definitons containing calls having the search term as their name.
  • Inspector You can navigate Object definitions and calls (and their associated nodes) just like any other JavaScript data structures. While inspecting an Object defintion, a button in the inspector header lets you edit that Object defintion. While inspecting a Call, a button in the inspector header lets you edit its containing Object definition, centering and selecting the Call block.

FPGA Type gives the HCA compiler a hint to optimize what it produces for different FPGA chips. This does not affect the HCA user interface in any other way.

Dataset Tree allows you to explore the numerous data types available. Clicking on one gives you a dialog box of operations on the selected Dataset

Object Tree allows you to explore the numerous object definitions available. Clicking on one gives you a dialog box of operations on the selected object. The most useful operation is making a call to that definition. The 2nd most useful operation is viewing (and editing) the definition itself.
HCA Editor Toolbar Left to right:
  • TreeGroup of the currently edited object definition.
  • ObjectName of the currently edited object definition.
  • Menu of all the defined sheets (top) and all the browsed (edited) non-sheet object definitions of this session. Choose one to edit it.
  • Sheet status of the edited object definition.

HCA Editor Operations
  • drag a call block to move it.
  • click a call block to select and inspect it.
  • shift-click on a block to add it to the selected blocks
  • ctrl-a select all blocks.
  • ctrl-x cut the selected blocks.
  • ctrl-c copy the selected blocks to the clipboard.
  • ctrl-v paste the selected blocks from the clipboard. This will adjust the suffixes on the names of the calls, i.e. ":A", ":B", etc. such that the call names in the object definitions remain unique.
  • drag an input or output circle to connect a wire to an output or input.
  • shift-click on an input or output to delete its transport (wire).
  • double-click on a block to show a dialog of operations.
  • 2 finger drag (or similar operation) on the editor pane to zoom (expand or shrink) the view.
Displayed Blocks that have a red background are calls to an object definition that is not defined in the current HCA_UI environment.
