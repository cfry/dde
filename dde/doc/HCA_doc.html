HCA User Interface
The HCA UI is a work in progress, not ready for use.
HCA stands for Heterogeneous Computing Architecture. HCA_UI is a user interface to writing software for FPGA's using a graphical editor. The tool creates a JSON representation of the layout of the objects in the editor. That JSON representation is sent to a compiler that makes a compiled gateware file which is stored on Dexter and controls the computation of the FPGA. That computation moves Dexter's joints and captures joint positions from the optical sensor on each joint.

HCA_UI reuses components of the DDE user interface for those opperations that are in common with normal DDE editing. When the HCA UI is brought up, it replaces the editor section of DDE's editor pane with a Boxes-and-Lines graph editor.

HCA editor operations
  • drag a node to move it.
  • click a node to select it.
  • ctrl-a select all nodes.
  • ctrl-x cut the selected nodes.
  • ctrl-c copy the selected nodes to the clipboard.
  • ctrl-v paste the selected nodes from the clipboard.
DDE Menu items for HCA
  • File menu/New Clears the canvas of all nodes and starts a "new buffer".
  • File menu/Open ... Allows the user to select a file from the computer's file system to edit.
  • File menu/Load ... Allows the user to select a file to load, which creates buttons of definitions of nodes. Clicking a button creates an instance of that definition, populating the canvas with nodes.
  • File menu/Save Saves the current file being edited (as a big JSON string).
  • File menu/Save as ... Lets the user choose a file to save the currently being edited nodes.
  • Edit menu/pretty print arranges the nodes to be well spaced.
Other DDE UI for HCA Files menu (select of recently used files) If you select a file without valid HCA code, it errors but leaves the current canvas and fikes menu unchanged.

Eval button
  • If no selection, inspect the HCA class itself.
  • If one selected node, inspect it.
  • If more than one selected node, inspect an array of those selected nodes.
